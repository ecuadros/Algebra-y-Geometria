Cambio de Base

El mismo vector puede describirse con coordenadas distintas dependiendo del sistema de referencia. La matriz de cambio de base traduce entre representaciones.

Base y coordenadas

Dado un espacio vectorial, una base B = {b₁, b₂} permite expresar cualquier vector v de forma única:

v = α·b₁ + β·b₂

[v]_B = (α, β) ← coords en base B
[v]_S = (x, y) ← coords estándar

Matriz de cambio de base

La matriz P cuyas columnas son los vectores base B convierte de coordenadas B a estándar:

P = [b₁ | b₂]

v = P · [v]_B (B → estándar)
[v]_B = P⁻¹ · v (estándar → B)

Propiedades

  • P es invertible ⟺ la base es linealmente independiente
  • Base ortonormal: P⁻¹ = Pᵀ (cálculo muy barato)
  • El cambio de base NO cambia el vector — solo su descripción
  • Composición: P₂⁻¹·P₁ lleva de base B₁ a base B₂

Presets de bases

Aplicación: Espacio local vs espacio mundial en motores de videojuegos

En Unity, Unreal o Godot cada objeto tiene un espacio local (local space) definido por su propia base {derecha, arriba, adelante}. Las coordenadas de los vértices de un mesh se guardan en espacio local. Para dibujarlos, el motor multiplica por la matriz de transformación del objeto — que no es otra cosa que la matriz de cambio de base de local a mundial (world space). Esto permite mover, rotar y escalar objetos sin recalcular la geometría del mesh.

El mesh del objeto (triángulo verde) se define en espacio local. La base actual determina cómo aparece en el mundo.
