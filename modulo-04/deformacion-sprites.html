Deformación de Sprites

El determinante de una matriz 2×2 cuantifica exactamente cómo una transformación lineal escala las áreas — y si invierte la orientación.

El Determinante 2×2

Para la transformación T(v) = Av, el determinante mide el factor por el cual se escala cualquier área bajo esa transformación:

A = [ a b ]
[ c d ]

det(A) = ad − bc

Área(T(S)) = |det(A)| · Área(S)

Interpretación Geométrica

  • det(A) > 0 → la transformación preserva la orientación
  • det(A) < 0 → la transformación invierte la orientación (espejo)
  • det(A) = 0 → las columnas son dependientes: la figura colapsa a una línea (o punto)
  • |det(A)| = 1 → transformación isométrica: no cambia el área
  • Las columnas de A son los vectores imagen de la base canónica

El Paralelogramo de las Columnas

Los vectores columna a₁ = [a, c]ᵀ y a₂ = [b, d]ᵀ son las imágenes de los vectores base e₁ y e₂. El paralelogramo que forman tiene exactamente área = |det(A)|.

a₁ = A·e₁ = [a, c]ᵀ
a₂ = A·e₂ = [b, d]ᵀ

Área del paralelogramo
= |det([a₁ | a₂])|
= |ad − bc|

Aplicación: Transformaciones de Sprites en Videojuegos

En motores de videojuegos 2D, cada sprite se almacena como un conjunto de píxeles en coordenadas locales. Para dibujarlo en pantalla se aplica una matriz de transformación 2×2 (más traslación). El determinante determina cuántos píxeles ocupa el sprite transformado: si |det| > 1 el sprite se agranda, si |det| < 1 se encoge, si det < 0 aparece invertido (espejo). Cuando det = 0 el sprite colapsa a una línea — un bug clásico de escala incorrecta.
